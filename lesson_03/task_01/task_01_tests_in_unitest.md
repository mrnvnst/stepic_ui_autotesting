### Задание: оформляем тесты в стиле unittest 

Оформляем тесты из первого модуля в стиле unittest.

1. Взять тесты из [этого шага](https://github.com/mrnvnst/stepic_ui_autotesting/tree/main/lesson_01/task_05/ "Переход к пятому уроку первого модуля 'Уникальность селекторов'").
2. Создать новый файл.
3. Создать в нем класс с тестами, который должен наследоваться от unittest.TestCase.
4. Переписать в стиле unittest [тест для первой страницы регистрации](http://suninjuly.github.io/registration1.html).
5. Переписать в стиле unittest [тест для второй страницы регистрации](http://suninjuly.github.io/registration2.html).
6. Оформить финальные проверки в тестах в стиле unittest, например, используя проверочный метод assertEqual.
7. Запустить получившиеся тесты из файла. 
8. Просмотреть отчёт о запуске.

По задумке должно выбрасываться исключение NoSuchElementException во втором тесте. Если была использована конструкция try/except, здесь нужно запустить тест без неё. Ловить исключения не надо (во всяком случае, здесь)!

Это всё для иллюстрации того, что unittest выполнит тесты и обобщит результаты даже при наличии неожиданного исключения.
